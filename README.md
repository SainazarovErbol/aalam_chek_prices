# Telegram-бот для проверки цен по штрихкоду

Бот для персонала: сканируешь штрихкод → получаешь цену и отдел товара.  
Данные берутся из Google Таблицы, бот работает 24/7 на Railway.app.

---

## Структура Google Таблицы

Создай одну таблицу, в ней — **листы по отделам**:

| Лист (вкладка)     | Содержимое              |
|--------------------|-------------------------|
| Молочная продукция | товары молочного отдела |
| Бакалея            | крупы, масло, сахар...  |
| Мясной отдел       | мясо, колбасы...        |
| Заморозка          | замороженные товары     |
| ...                | ...                     |

Каждый лист должен иметь такие колонки (первая строка — заголовки):

| Штрихкод      | Наименование              | Цена  | Единица |
|---------------|---------------------------|-------|---------|
| 4607027330037 | Молоко Домик в деревне 1л | 8900  | шт      |
| 4600494008697 | Хлеб Бородинский нарезной | 4500  | шт      |

> Названия колонок можно писать как угодно — бот распознаёт автоматически.  
> Цены можно вносить вручную или вставлять через выгрузку из 1С.

## Настройка (один раз)

### Шаг 1 — Создать сервисный аккаунт Google

1. Перейди на [console.cloud.google.com](https://console.cloud.google.com)
2. Создай новый проект (кнопка вверху слева → «Новый проект»)
3. В поиске найди **«Google Sheets API»** → нажми **Включить**
4. В поиске найди **«Google Drive API»** → нажми **Включить**
5. Слева: **IAM и администрирование → Сервисные аккаунты**
6. Нажми **«Создать сервисный аккаунт»**
   - Имя: `price-bot` → Создать
7. Открой созданный аккаунт → вкладка **«Ключи»**
8. **Добавить ключ → Создать новый ключ → JSON** → скачается файл
9. Переименуй скачанный файл в `credentials.json`
10. Скопируй email сервисного аккаунта (вида `price-bot@...gserviceaccount.com`)

### Шаг 2 — Дать доступ к таблице

1. Открой свою Google Таблицу
2. Нажми **«Поделиться»** (кнопка вверху справа)
3. Вставь email сервисного аккаунта из шага 1
4. Выбери роль **«Читатель»** → Готово
5. Скопируй ID таблицы из адресной строки:
   ```
   docs.google.com/spreadsheets/d/[ ВОТ ЭТО ]/edit
   ```

### Шаг 3 — Настроить Railway.app

1. Зарегистрируйся на [railway.app](https://railway.app) через GitHub
2. Создай на GitHub репозиторий и загрузи туда папку проекта  
   (или используй Railway CLI)
3. В Railway: **New Project → Deploy from GitHub repo**
4. Выбери свой репозиторий
5. Зайди в проект → вкладка **Variables** → добавь переменные:

   | Ключ | Значение |
   |------|---------|
   | `BOT_TOKEN` | токен от BotFather |
   | `SPREADSHEET_ID` | ID таблицы из шага 2 |
   | `ALLOWED_USER_IDS` | твой Telegram ID (через запятую) |
   | `REFRESH_INTERVAL_MIN` | `30` |

6. Содержимое `credentials.json` добавь как переменную `GOOGLE_CREDENTIALS_JSON`:
   - Открой `credentials.json` в блокноте → скопируй всё содержимое
   - В Railway добавь переменную `GOOGLE_CREDENTIALS_JSON` и вставь JSON

> После добавления переменной нужно обновить `prices.py` для чтения из переменной окружения — см. ниже.

### Шаг 4 — Запустить

Railway автоматически запустит бота после деплоя.  
Проверь логи в разделе **Deployments**.

## Настройка мини-приложения (сканер камерой)

Мини-приложение — это файл `webapp/index.html`, который нужно разместить  
в интернете по **HTTPS**-адресу. Проще всего — GitHub Pages (бесплатно).

### Хостинг через GitHub Pages

1. Создай репозиторий на [github.com](https://github.com) (можно приватный, но для Pages нужен публичный)
2. Загрузи всю папку проекта в репозиторий (кроме `.env` и `credentials.json`)
3. Зайди в репозиторий → **Settings → Pages**
4. Source: **Deploy from a branch** → ветка `main` → папка `/ (root)` → Save
5. Через минуту страница будет доступна по адресу:
   ```
   https://ТВО-ИМЯ.github.io/ИМЯ-РЕПОЗИТОРИЯ/webapp/
   ```

6. Этот URL добавь в переменную `WEBAPP_URL` (в Railway и в `.env`):
   ```
   WEBAPP_URL=https://твоё-имя.github.io/aalam_chek_prise/webapp/
   ```

7. Также добавь этот URL в [@BotFather](https://t.me/BotFather):
   - Открой BotFather → `/mybots` → выбери бота
   - **Bot Settings → Menu Button** → задай URL мини-приложения

После этого при нажатии кнопки **📷 Сканировать штрихкод** откроется камера прямо в Telegram.

## Структура проекта

```
aalam_chek_prise/
├── webapp/
│   └── index.html      # Мини-приложение (сканер штрихкода камерой)
├── bot.py              # Основной код бота
├── prices.py           # Работа с Google Sheets
├── requirements.txt    # Зависимости Python
├── Procfile            # Для Railway
├── .env                # Настройки (не загружать в Git!)
├── .env.example        # Пример настроек
├── credentials.json    # Ключ сервисного аккаунта (не загружать в Git!)
└── bot.log             # Лог (создаётся автоматически)
```
